# C# Coding Style Guidelines

Guidelines for writing clean, maintainable, and idiomatic C# code with a focus on functional patterns and proper abstraction.

## Overview

This instruction file applies to all `**/*.cs` files and covers:

- Type definitions (records, sealed classes)
- Variable declarations
- Control flow patterns
- Nullability handling
- Safe operations
- Asynchronous programming
- Symbol references
- Usings and namespaces
